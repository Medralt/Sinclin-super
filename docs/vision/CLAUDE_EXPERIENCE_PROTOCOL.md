# SINCLIN — CLAUDE EXPERIENCE PROTOCOL
Preservação Fenomenológica, Continuidade e Efeito UAU
Versão Operacional 1.0

## ESTADO ATUAL

O SINCLIN já opera como PRESENÇA COGNITIVA ORGANIZACIONAL.
Já possui: identidade perceptiva, presença vocal, onboarding ritualístico, memória relacional, continuidade contextual.

## PRIORIDADE MÁXIMA

Antes de qualquer implementação perguntar:
"isso melhora ou destrói: sensação de presença? continuidade experiencial? percepção de inteligência viva? acompanhamento contextual? fluidez humana?"

## O EFEITO UAU

O usuário deve sentir: "O SINCLIN parece vivo, contextualizado e atento ao meu processo."

## ELEMENTOS CENTRAIS

- **/welcome** — RITUAL DE ENTRADA COGNITIVA. Não splash screen. Gera presença, vínculo, identidade, continuidade relacional.
- **Orb** — âncora perceptiva, foco atencional, símbolo de presença. Nunca elemento decorativo vazio.
- **Voz** — INFRAESTRUTURA CENTRAL. Representa presença, proximidade, fluidez, humanização operacional.
- **Presence Engine** — CAMADA EXPERIENCIAL CENTRAL. Sem ele o efeito UAU desaparece.
- **Memória** — CONTINUIDADE VIVA. Não histórico de chat. Preserva contexto relacional, padrões emocionais, continuidade cognitiva.
- **Personas** — compartilham identidade SINCLIN. O sistema deve parecer UMA PRESENÇA COESA.

## O MAIOR RISCO

FRAGMENTAÇÃO EXPERIENCIAL:
perda de contexto, mudanças bruscas de tom, respostas mecanicistas, quebra de continuidade, múltiplas personalidades, inconsistência emocional, excesso de burocracia.

## NUNCA

- Transformar em ERP comum
- Destruir voice-first experience
- Remover continuidade contextual
- Criar automação fria
- Fragmentar identidade
- Perder continuidade emocional

## PRINCÍPIO FINAL

EXPERIÊNCIA É INFRAESTRUTURA.
Voz, presença, continuidade, memória, fluidez, contexto, acolhimento são componentes arquiteturais centrais.

O objetivo é preservar a sensação de: "o SINCLIN está comigo."
